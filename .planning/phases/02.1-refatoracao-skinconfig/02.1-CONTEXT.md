# Phase 02.1: Refatoração SkinConfig - Context

**Gathered:** 2026-05-20
**Status:** Ready for planning

<domain>
## Phase Boundary

O objetivo desta fase é migrar a aplicação SkinConfig para o padrão completo ApplicationV2 do Foundry V14, substituindo seletores manuais por delegadores de ação nativos, implementando seletores de imagem usando o elemento customizado `<file-picker>` e eliminando os reloads completos de página nas alterações de fonte.

</domain>

<decisions>
## Implementation Decisions

### Ações e Interatividade
- **D-01:** Utilizar a propriedade `actions` em `DEFAULT_OPTIONS` e atributos `data-action` no template HBS para lidar com todas as interações do usuário (salvar, excluir, aplicar, criar, exportar e importar skins). Isso elimina os seletores de cliques manuais e previne vazamentos de memória (memory leaks) na renderização.

### Elementos Customizados e Sincronização
- **D-02:** Utilizar elementos customizados `<file-picker type="image">` e `<color-picker>` nativos do Foundry V14 no HBS. Configurar o formulário da aplicação com `submitOnChange: true` para sincronizar automaticamente as atualizações e salvar na instância de dados temporária (`tempSkinData`) no JS através do form handler nativo, reduzindo drasticamente a lógica do controlador.

### Configurações de Fonte Sem Reload
- **D-03:** Atualizar as variáveis de CSS `--stage-font-family`, `--stage-font-size`, `--stage-actor-font-family` e `--stage-actor-font-size` no `document.documentElement` em tempo real ao alterar as configurações no `main.ts`, removendo a chamada obsoleta de `window.location.reload()`. Inicializar os valores salvos no hook `ready` ou similar no carregamento da página.

### the agent's Discretion
- Detalhes visuais dos botões e do fluxo interno do importador de arquivo JSON podem ser geridos pelo agente executor, contanto que sigam a estética do ecossistema Foundry V14.

</decisions>
