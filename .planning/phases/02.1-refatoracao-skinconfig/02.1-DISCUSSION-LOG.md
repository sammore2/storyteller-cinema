## Sistema de Ações no ApplicationV2

| Option | Description | Selected |
|--------|-------------|----------|
| Opção A | Ações nativas declarativas via `DEFAULT_OPTIONS.actions` + `data-action` no template HBS. | ✓ |
| Opção B | Delegação de eventos manuais usando cliques ad-hoc no `_onRender`. | |

**User's choice:** Opção A.
**Notes:** O usuário concordou em consolidar a migração completa, eliminando os padrões de JQuery/DOM manual herdados do V13.

## Elementos Customizados do V14

| Option | Description | Selected |
|--------|-------------|----------|
| Opção A | Substituição de inputs manuais por `<file-picker type="image">` e `<color-picker>` nativos do Foundry V14 no HBS com sincronização via formulário (`submitOnChange: true`). | ✓ |
| Opção B | Manter inputs manuais e instanciamento manual via JS de classes como `FilePicker`. | |

**User's choice:** Opção A.
**Notes:** O usuário validou a remoção de código de sincronização manual, aderindo inteiramente às novidades de web components do Foundry V14.

## Configurações de Fonte Sem Reload

| Option | Description | Selected |
|--------|-------------|----------|
| Opção A | Modificar variáveis de CSS Custom Properties no `:root` em tempo real via callback `onChange`. | ✓ |
| Opção B | Manter a chamada de `window.location.reload()`. | |

**User's choice:** Opção A.
**Notes:** O usuário optou por remover reloads de página inteira que prejudicavam a fluidez do jogo em favor de uma UX instantânea.
